## out_c

Plain C output, generated and compile-checked with `gcc` on this machine
(no GPU/CUDA toolchain available here). Not wired into the OpenFOAM
`kodesChemistryModel` library, which expects CUDA (`.cuh`/`.cu`) sources
like `pyJacSource/grimech/out` — regenerate with `-l cuda` on a machine
with `nvcc` to produce that variant (see below).

Regenerate:

```bash
source venv/bin/activate
MECH_DIR=external/Kinetic-Mechanisms/Gas-Phase/CoreMechanism_C0-C4/Soot-NOx/C1_C3_HT_NOX_159_2459
python -m pyjac -l c \
  -i "$MECH_DIR/kinetics.CHEMKIN.CKI" \
  -t "$MECH_DIR/thermo.CHEMKIN.CKT" \
  -b src/kodesChemistyModel/pyJacSource/C1_C3_HT_NOX_159_2459/out_c

# for the GPU/CUDA variant, on a machine with nvcc:
python -m pyjac -l cuda \
  -i "$MECH_DIR/kinetics.CHEMKIN.CKI" \
  -t "$MECH_DIR/thermo.CHEMKIN.CKT" \
  -b src/kodesChemistyModel/pyJacSource/C1_C3_HT_NOX_159_2459/out
```

## pyJac fork fixes

Generating this mechanism's analytical Jacobian hit two bugs in the PLOG
(pressure-dependent Arrhenius) code path of
`external/pyJac/pyjac/core/create_jacobian.py::write_plog_rxn_dt`, triggered
because this mechanism has 236 PLOG reactions (1291 pressure-bracket
entries) — far more than the fork's original test mechanisms (grimech,
h2o2) which have none. Both were fixed directly in the pyJac submodule:

1. **All-zero pressure-bracket segment** (6 segments, e.g. reaction 409
   `H + C3H6 <=> CH3 + C2H4`): when both bracketing pressure points have
   `b == 0` and `E == 0` (a purely T-independent rate), the code asserted
   `"PLOG Derivative undefined"` instead of emitting the (mathematically
   correct) zero contribution. Fixed to emit an explicit `0.0e0)` term.
2. **b-only pressure-bracket segment** (2 segments, e.g. reaction 467):
   when only the temperature exponent `b` differs between the two
   bracketing points (activation energy `E` identical), the code left a
   dangling `+` with no closing paren, because it only closed/emitted the
   log-pressure interpolation weight from inside the `E`-difference
   branch. Restructured so the weight is always emitted once, regardless
   of which of `b`/`E` actually differs.

Both were verified by compiling every generated `.c` file with `gcc -O0`
(this mechanism has no reachable negative-`A` or equal-pressure PLOG
segments — checked separately).
